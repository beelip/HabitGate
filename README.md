# HabitGate

HabitGate は、毎日の「やること」と「減らすこと」を記録する Android アプリです。  
指定時刻に「今日は何をした？」という通知を出し、タスク消化・時間・メモ・一日のサイクルを記録できます。

## ダウンロード

Androidスマホに直接インストールする場合は、以下からAPKをダウンロードしてください。

[APKをダウンロード](https://github.com/beelip/HabitGate/raw/refs/heads/main/build/habit-gate.apk)

※ Playストア経由ではないため、インストール時に「不明なアプリのインストールを許可」が必要になる場合があります。

## 機能

### やること

- 「タスク編集」画面から、「今日やる」「明日やる」またはカレンダーで選んだ任意の日付に、やることを追加できます。追加フォームでも優先度・期限を入力できます（期限ボタンの長押しでクリア）。
- 各項目にメモ・優先度（なし/低/中/高）・期限（日時）を設定できます。優先度に応じて行の背景色が変わります（高=赤 / 中=黄 / 低=緑）。
- 実績入力画面でも、予定日・優先度・期限をその場で変更できます（変更は即時保存されます）。
- トップ画面の「やること」は「優先順位（手動並べ替え）／優先度／期限／予定日／作成日」から並び順を選べます。
  - 「期限」ソートでは「今日やること」「明日以降やること」に分かれて表示され、見出しをタップすると折りたたみできます。
  - 「優先順位」ソート中は、項目を長押ししてドラッグすると手動で並べ替えられます。
- トップ画面でタスクをタップすると、個別の実績入力画面が開きます。
  - 入力日を変更できます（既定は現在の対象日）。
  - タスク作成時のメモをその場で編集できます（メモ行または 📝 をタップ。変更は即時保存されます）。
  - 実績時間・実績メモ（長文可）を記録できます。
  - 「完了する」にチェックして保存すると、確認のうえタスクを完了にします。
  - 「明日に繰り越す」ボタンから、その場でタスクを翌日へ移動できます。
- 実績時間だけ追加して、タスクは未完了のまま残すこともできます。
- 完了したタスクは一覧から消え、集計画面の「完了したタスク」に表示されます（同名タスクの実績時間は完了日ごとに正しく按分されます）。
- トップ画面の「→」ボタンで、タスクを翌日へ繰り越せます。
- 未完了タスクは、一日を終了したときにも次の日へ繰り越されます。
- 「タスク編集」画面では、項目をタップすると実績入力画面、長押しすると名前・メモ・日付・優先度・期限を編集する画面が開きます。削除もこの画面から行えます（確認ダイアログあり）。

### 減らすこと

- 「タスク編集」画面から「減らすこと」を登録できます。
- 各項目にメモを付けられます。トップ画面で長押しドラッグして並べ替えられます。
- トップ画面で項目をタップすると、個別の実績入力画面が開きます。入力日の変更、時間・実績メモの記録ができます。
- 「タスク編集」画面の 📱 からアプリを連携すると、そのアプリの使用時間を自動計測できます（アイコン表示・名前検索付き）。連携項目には手動で時間を追加入力する欄もあります。
- アプリ連携中の使用時間は、保存前でもリアルタイムに集計へ反映されます。計測分は一日の終了時に自動保存され、重複して保存されることはありません。
- トップ画面に使用時間バーを表示します（〜10%青 / 〜25%緑 / 〜50%黄 / それ以上赤、100%超で全体赤）。上限は項目ごとに実績入力画面から設定でき、既定は8時間です。
- 「タスク編集」画面では、項目をタップすると実績入力画面、長押しすると名前・メモ・連携アプリを編集する画面が開きます。

### サイクル

- アプリ内の「一日」は、現在のサイクルとして管理されます。
- サイクルには対象日・開始日時・終了日時が記録されます。
- 通知時刻を待たずに、手動で「この日を終わらせて次の日へ」移れます。
- 設定した時刻（既定 5:00）を過ぎると、サイクルは自動で終了して次の日へ移ります。端末再起動後も自動終了は引き継がれます。
- 一日を終了すると、未完了のやることは次の対象日に繰り越されます。
- データ変更のたびに、設定済みの CSV とスプレッドシートへ自動で同期します。

### CSV バックアップ / 移行

設定画面の「CSVバックアップ / 移行」から次の操作ができます。

- `CSVを出力・共有`
  - 現在の全データをダウンロードフォルダへ出力し、共有できます。
- `CSV更新先フォルダを選択`
  - Android のフォルダ選択画面を開きます。
  - 選択したフォルダに `habit_gate_data.csv` を作成・更新します。
- `指定フォルダのCSVを今すぐ更新`
  - 現在の全データを即時エクスポートします。
- `CSVからインポート`
  - バックアップCSVを読み込み、アプリ内データをCSVの内容で置き換えます。

CSVには以下のセクションが含まれます。

```text
# habit_gate_csv_v1
# do_tasks
# reduce_items
# records
# cycles
```

インポートは「追記」ではなく「置き換え」です。移行前には、念のため現在のデータをCSV出力してください。

### 集計

- 今日・今週・今月・任意の日付の実績を表示できます。ピル型トグルのほか、画面のスワイプでも切り替えられます。
- 📅 のカレンダーから過去の任意の日・週・月を選んで表示できます。選択した日付に応じて、チップの名称が「今週」→「この週」のように変化します。
- 「やること」と「減らすこと」の時間を棒グラフで表示します。今週は常に7日分、今月は週別の内訳と合計を棒グラフで表示します。
- 完了したタスクの一覧（完了日・実績時間）を確認できます。
- 実績一覧にはメモを表示し、実績時間は右揃えで見やすく表示します。実績をタップすると、入力日・時間・メモの編集や削除ができます。
- サイクル履歴として、開始日時・終了日時も確認できます。

### デザイン

- 設定画面の「デザイン」から、30種類のカラーテーマを選べます。
- ダークモードのオン/オフを切り替えられるほか、「端末のダークモードと連携する」を有効にすると端末側の設定に合わせて自動で切り替わります。
- テーマは全画面（背景・カード・ボタン・ダイアログ・ステータスバーなど）に反映されます。

### スプレッドシート連携

- Google Apps Script の Web App URL を設定画面のダイアログから設定すると、未同期データを Google Sheets に送信します。
- 実績は `records` シートへ送信されます。
- 終了済みサイクルは `cycles` シートへ送信されます。
- CSVバックアップとは独立して動作します。

Apps Script のサンプルは以下にあります。

```text
tools/google-sheets-webhook.gs
```

## GitHub Actions で APK をビルドする

1. このプロジェクトを GitHub リポジトリへ push します。
2. GitHub の `Actions` タブを開きます。
3. `Build Android APK` を実行します。
4. Artifact の `habit-gate-apk` をダウンロードします。
5. 中にある `habit-gate.apk` を Android 端末へインストールします。

## 上書きアップデート時の注意

既存データを残すには、必ず同じアプリとして上書きインストールしてください。

- `applicationId` は変更しないでください。
- Android では、上書きアップデート時に前回 APK と今回 APK の署名キーが一致している必要があります。
- この版でも `signing/habitgate-debug.keystore` を同梱し、GitHub Actions の debug APK で毎回同じ署名になるようにしています。
- 既存アプリをアンインストールすると、端末内のアプリデータも削除される場合があります。
- CSV エクスポート / インポートで移行できます。
- データベースは起動時に自動でマイグレーションされます（v0.5.0 で `reduce_items` に `app_package` 列、v0.6.0 で `do_tasks` に `completed` / `completed_date` 列、`reduce_items` に `gauge_max_minutes` 列、v0.8.0 で `do_tasks` に `sort_order` / `priority` / `due_at` 列、`reduce_items` に `sort_order` 列が追加されます）。

## Android の通知制限について

Android のバージョンや端末設定によっては、通知時刻に完全なポップアップ表示ができないことがあります。  
その場合は高優先度通知をタップして入力画面を開いてください。

アプリ初回起動後、必要に応じて以下を許可してください。

- 通知権限
- 正確なアラーム権限
- 使用状況へのアクセス（アプリ使用時間の自動計測を使う場合）
