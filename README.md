# HabitGate

HabitGate は、毎日の「やること」と「減らすこと」を記録する Android アプリです。
指定時刻に「今日は何をした？」という通知を出し、タスク消化・時間・メモを記録できます。

## この版について

version `0.2.0` では、既存データを保持したアップデートを前提にしています。

- `applicationId` は `com.habitgate.app` のままです。
- 既存DB名 `friction_habit.db` は、データ保持のためそのまま維持しています。
- DBバージョンを `1` から `2` に上げ、既存テーブルへ `note` 列を追加します。
- 新しく `cycles` テーブルを追加し、一日の開始日時・終了日時を保存します。

そのため、同じ `applicationId` の APK として上書きインストールすれば、既存のタスク・実績データを残したまま移行されます。

## 機能

### やること

- 「当日」または「次の日」のやることを追加できます。
- 各項目にメモを付けられます。
- 入力画面は通知時刻に限らず、いつでも開けます。
- 各タスクについて、次の操作ができます。
  - 実績時間を追加する
  - 完了する
  - 実績メモを残す
- 実績時間だけ追加して、タスクは未完了のまま残すこともできます。
- 完了したタスクは一覧から消えます。
- 未完了タスクは、手動で一日を終了したときに次の日へ繰り越されます。

### 減らすこと

- 事前に「減らすこと」を登録できます。
- 各項目にメモを付けられます。
- その日にやってしまった項目に対して、時間と実績メモを記録できます。

### サイクル

- アプリ内の「一日」は、現在のサイクルとして管理されます。
- サイクルには対象日・開始日時・終了日時が記録されます。
- 通知時刻を待たずに、手動で「この日を終わらせて次の日へ」移れます。
- 一日を終了すると、未完了のやることは次の対象日に繰り越されます。

### 集計

- 今日・今週・今月の実績を表示できます。
- 「やること」と「減らすこと」の時間を棒グラフで表示します。
- 実績一覧にはメモと同期状態も表示されます。
- サイクル履歴として、開始日時・終了日時も確認できます。

### スプレッドシート連携

- Google Apps Script の Web App URL を設定すると、未同期データを Google Sheets に送信します。
- 実績は `records` シートへ送信されます。
- 終了済みサイクルは `cycles` シートへ送信されます。
- 連携前でも CSV 出力が可能です。

Apps Script のサンプルは以下にあります。

```text
tools/google-sheets-webhook.gs
```

## GitHub Actions で APK をビルドする

1. このプロジェクトを GitHub リポジトリへ push します。
2. GitHub の `Actions` タブを開きます。
3. `Build Android APK` を実行します。
4. Artifact の `HabitGate-debug-apk` をダウンロードします。
5. 中にある `app-debug.apk` を Android 端末へインストールします。

## 上書きアップデート時の注意

既存データを残すには、必ず同じアプリとして上書きインストールしてください。

- `applicationId` は変更しないでください。
- Android では、上書きアップデート時に前回 APK と今回 APK の署名キーが一致している必要があります。
- この版からは `signing/habitgate-debug.keystore` を同梱し、GitHub Actions の debug APK でも毎回同じ署名になるようにしています。
- 既存アプリをアンインストールすると、端末内のアプリデータも削除される場合があります。
- 念のため、アップデート前に旧版から CSV 出力しておくことを推奨します。

### 旧版を GitHub Actions の一時 debug 署名で入れていた場合

以前の APK が GitHub Actions の自動生成 debug キーで署名されていた場合、今回の APK と署名が一致せず、Android が上書きインストールを拒否することがあります。
その場合、アプリ側の `applicationId` と DB 移行処理だけではデータを直接引き継げません。

デバッグビルドであれば、PC から `adb` と `run-as` を使って旧DBを退避し、新版インストール後に戻す方法があります。
不安な場合は、まず旧版で CSV 出力してから作業してください。

## Android の通知制限について

Android のバージョンや端末設定によっては、通知時刻に完全なポップアップ表示ができないことがあります。
その場合は高優先度通知をタップして入力画面を開いてください。

アプリ初回起動後、必要に応じて以下を許可してください。

- 通知権限
- 正確なアラーム権限
